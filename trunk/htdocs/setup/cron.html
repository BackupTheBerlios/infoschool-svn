Für manche kommt jetzt der unangenehme Teil. Wem cron zur Verfügung steht (unter Linux Standard) füllt einfach das Formular aus:

HTTP-Host1):

crontab direkt eintragen2)

1) Der HTTP-Host ist der Name des Servers, der erreicht werden soll. Um es kompliziert zu machen, muss man korrekterweise in diesem Fall sagen, dass es der Name ist, auf den der Server antworten soll.
In den meisten Fällen reicht hier der Domainname aus. Im Falle eines virtuellen Servers, muss hier der Name des virtuellen Servers eingetragen werden.

2) Diese Option ist nur in seltenen Fällen sinnvoll. Das Setup wird die Datei config/crontab erzeugen, in der steht, wann was ausgeführt werden soll.
Wenn man diese Option nun aktiviert, werden diese Informationen automatisch in die Crontabelle des Benutzers eingetragen, mit dem der Webserver läuft. Dabei werden alle vorigen Informationen aus der Tabelle überschrieben. Also Vorsicht, wenn der Webserver von mehreren Personen benutzt wird.
Der Befehl crontab pfad/zu/infoschool/config/crontab überschreibt die Crontabelle des aktuell eingeloggten Benutzers. Wenn kein Shell-Zugriff zur Verfügung steht, muessen manche Skripte manuell gestartet werden.
Unter Windows kann man die Informationen aus der Datei config/crontab adäquat in den Taskmanager eintragen.
